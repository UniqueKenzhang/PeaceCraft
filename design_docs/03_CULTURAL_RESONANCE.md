# 模块四：逃兵与文明共振系统 (Deserter & Cultural Resonance)

这是游戏的高阶玩法和核心正反馈循环。它将玩家在战场上的“非致命”策略（使用恶搞物资）转化为长期的科技优势，是通往高级策略的必经之路。

## 1. 核心任务与目标

- **回收人口**：建立一个机制，用于接收和管理通过“恶搞物资”从战场上“截留”下来的“逃兵”。
- **转化价值**：将“逃兵”转化为两类核心价值：特殊的劳动力，以及用于科技研发的“科研点数”。
- **驱动创新**：通过“文明共振”产生的科研点数，让玩家可以研发更高级的“恶搞物资”，形成战略闭环。

---

## 2. 功能拆解与实现方案

### 任务 4.1：逃兵人口管理

- **逃兵 (Deserter) 的产生**:
    - **唯一来源**: 逃兵的数量和阵营归属，完全由“战场与动态平衡系统”（模块一）在每次结算后输出。
    - **计算公式 (在模块一中实现)**: `产生的逃兵数 = (预期总伤亡) * DeserterRateModifier`
    - **数据接收**: 本模块需要一个接口 `AddDeserters(allianceDeserters, abyssDeserters)` 来接收新产生的逃兵。

- **逃兵人口池**:
    - **数据结构**:
        - `TotalAllianceDeserters` (整数): 联盟方逃兵总人口。
        - `TotalAbyssDeserters` (整数): 深渊方逃兵总人口。
        - `AssignedDeserters` (整数): 已被分配去工作的逃兵数量。
        - `IdleDeserters` (整数): 空闲的逃兵数量。
    - **作为劳动力**:
        - 逃兵可以像“Anno风格建设系统”（模块三）中的普通人口一样，被视为劳动力。
        - **特殊加成**: 可以为逃兵设计一些区别于普通民众的特性，例如：
            - **正面**: 在特定岗位上工作效率更高（例如，深渊逃兵在矿场工作有加成）。
            - **负面**: 需要一些特殊的需求才能维持满意度（例如，需要“地下酒馆”等特殊建筑）。

### 任务 4.2：文明共振与科技研发

- **文明共振 (Cultural Resonance) 效应**:
    - **核心理念**: 两种不同文明（联盟/深渊）的知识在玩家的庇护下交融，产生创新的火花。
    - **科研点数 (Research Points) 生成**:
        - **生成速率**: `RP_per_minute = BaseRate * TotalDeserters * BalanceFactor`
        - **`TotalDeserters`**: `TotalAllianceDeserters + TotalAbyssDeserters`。逃兵总数越多，基础产出越高。
        - **`BalanceFactor`**: `1 - abs(A - B) / (A + B)`，其中 A 和 B 分别是两方逃兵的数量。这个因子用于奖励玩家维持两方逃兵数量的均衡。当双方数量完全相等时，`BalanceFactor` 为1，产出效率最高；当一方数量远超另一方时，因子趋近于0，产出效率大打折扣。
        - **示例**:
            - 100联盟兵, 100深渊兵: `BalanceFactor` = 1.0 (效率100%)
            - 150联盟兵, 50深渊兵: `BalanceFactor` = 0.5 (效率50%)

- **联合实验室与科技树**:
    - **UI 设计**:
        - 创建一个名为“联合实验室”或“地下工坊”的科技研发界面。
        - 界面上方需要清晰展示当前“科研点数”的总量和每分钟的产出速率。
    - **科技树 (Tech Tree)**:
        - 科技树上的节点是各种待解锁的项目。
        - **主要内容**:
            - **新型恶搞物资**: 解锁新的、效果更强或更奇特的恶搞物资生产配方。例如：“战场迪斯科舞厅”、“便携式充气城堡”等。
            - **效率提升**: 提升现有恶搞物资的生产效率或效果。
            - **特殊建筑**: 解锁用于满足逃兵特殊需求的建筑，或进一步提升科研点数产出的建筑。
        - **解锁方式**: 玩家消耗积累的“科研点数”来解锁科技节点。

---
## 3. 关联模块接口

- **输入**：
    - `(来自模块一)` 接收 `AddDeserters` 指令，获得新逃兵的数量和阵营。
- **输出**：
    - `(至模块三)` 将“逃兵”作为可用劳动力注入到经济系统中。
    - `(至模块三)` 当新的“恶搞物资”配方被研发出来后，将其添加到对应的生产建筑的可用配方列表中。
    - `(至UI系统)` 提供科研点数、产出速率、逃兵数量等数据用于界面展示。
