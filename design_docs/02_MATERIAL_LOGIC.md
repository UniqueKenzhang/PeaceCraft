# 模块二：三色物资与交付系统 (Material & Delivery System)

该模块是玩家与战场交互的核心，玩家通过生产和交付不同性质的物资来精细地操纵战局，实现“动态平衡”的战略目标。

## 1. 核心任务与目标

- **定义物资**：明确“真货”、“假货”、“恶搞物资”三种类型的属性和游戏内效果。
- **实现交付**：提供玩家向“大世界意志”交付物资的交互界面和功能。
- **连接战场**：将物资交付行为准确地转化为对“战场与动态平衡系统”的输入。

---

## 2. 功能拆解与实现方案

### 任务 2.1：三色物资的定义与效果

所有物资都应作为游戏内的“物品(Item)”存在，拥有名称、图标、描述等基本属性，并可被存储在仓库中。其核心区别在于效果参数。

- **真货 (True Armaments)**
    - **效果描述**：“为前线注入真正的力量，但代价是惨烈的伤亡。”
    - **核心参数**:
        - `BalanceImpact` (正值, e.g., +10 per unit): 对战场天平的直接影响值。
        - `FatalityRateModifier` (正值, e.g., +5%): 对战场“阵亡率”的修正系数。
        - `DeserterRateModifier` (负值, e.g., -5%): 对战场“逃兵”产出率的修正系数。
    - **实现逻辑**：交付后，其 `BalanceImpact` 值将直接增加目标势力的 `BalanceValue`。同时，其修正系数会影响当次战场结算的伤亡和逃兵计算。

- **假货 (Counterfeit Gear)**
    - **效果描述**：“劣质的装备会拖累军队，但这种欺骗可能被识破。”
    - **核心参数**:
        - `BalanceImpact` (负值, e.g., -10 per unit): 对战场天平的直接影响值（削弱目标势力）。
        - `FatalityRateModifier` (正值, e.g., +2%): 同样会引发伤亡，但程度可能低于真货。
        - `DetectionRisk` (概率, e.g., 15% per delivery): 每次交付被“大世界意志”识破的风险。
    - **实现逻辑**：交付后，其 `BalanceImpact` 值将直接**降低**目标势力的 `BalanceValue`。同时，系统需进行一次随机判定，若判定失败（被识破），则立即触发一次性的严厉惩罚（来自模块一）。

- **恶搞物资 (Whimsical Subversion)**
    - **效果描述**：“通过非致命的恶作剧让战局看起来很激烈，从而在保存有生力量的同时满足上意。”
    - **核心参数**:
        - `BalanceImpact` (正值, e.g., +8 per unit): 同样能增强目标势力的“表现力”，但效果略低于真货。
        - `FatalityRateModifier` (极大的负值, e.g., -50%): 极大地降低战场“阵亡率”。
        - `DeserterRateModifier` (极大的正值, e.g., +50%): 极大地提高“逃兵”的产出率。
    - **实现逻辑**：交付后，其 `BalanceImpact` 值增加目标势力的 `BalanceValue`。最关键的是，它将显著改变当次战场结算的公式，将大量的“预期阵亡人口”转化为“逃兵人口”，并传递给模块四。

### 任务 2.2：交付与结算逻辑

- **UI 设计**：
    - 创建一个“征集令”或“上缴”界面。
    - 该界面清晰地展示当前“大世界意志”的指令（来自模块一），包括所需物资、数量和时限。
    - 玩家可以在此界面从自己的仓库中选择物资并点击“交付”按钮。系统需提供数量选择器和清晰的库存显示。

- **结算函数 `SettleDelivery(deliveryInfo)`**：
    - **触发时机**：玩家点击“交付”按钮时调用。
    - **输入参数 `deliveryInfo`**：包含所交付物资的列表（类型和数量）以及目标势力。
    - **执行流程**:
        1. 检查玩家仓库中是否有足够的物资。
        2. 扣除玩家仓库中的相应物资。
        3. 遍历交付的物资列表，累加计算总的 `BalanceImpact`, `FatalityRateModifier`, `DeserterRateModifier`。
        4. 对每一次“假货”交付，执行 `DetectionRisk` 判定。如果失败，立即调用模块一的惩罚函数并中断或削弱本次交付的效果。
        5. 调用模块一的接口 `UpdateBalance(totalBalanceImpact, totalFatalityModifier, totalDeserterModifier)`，将计算好的总影响值传递给战场系统。
        6. 更新“征集令”的状态（例如，更新已交付数量）。

---
## 3. 关联模块接口

- **输入**：
    - `(来自模块一)` 接收当前的“征集令”数据以供UI展示。
    - `(来自模块三)` 查询玩家仓库中的物资库存。
- **输出**：
    - `(至模块一)` 调用 `UpdateBalance` 接口，将物资交付产生的综合影响传递给战场。
    - `(至模块三)` 交付成功后，通知经济系统扣除仓库中的物资。
